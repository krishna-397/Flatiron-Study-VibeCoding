digital object 03 — the network that built the flatiron

A building is a relational structure before it is a physical one. This graph, built in D3.js from two CSV files (nodes + edges), maps the network that produced the Flatiron: the grid and the diagonal that manufactured its lot, the developer, architects, and engineers who shaped it, the firms that fabricated its systems, and the press and photographers who turned a mocked office block into an icon. Line weight encodes the strength of each relationship; arrows show direction; node size and tone encode each actor's kind. The data lives in flatiron-nodes.csv and flatiron-edges.csv, following the course template's exact format.

RELATIONAL STRUCTURES IN MY OWN WORK

The lesson I take from this object is that a relational structure can be a generator, not just a description. My interests run through spatial layout at several scales — furniture in rooms, rooms in plans, parcels on farmland — and each of these is a graph before it is a drawing: rooms are nodes and required adjacencies are edges (kitchen–dining strong, bedroom–street weak), exactly as crops are nodes and rotation compatibilities are edges. A force layout like this one is already a primitive floor-plan generator: let the edges pull what must be adjacent, let the charge push apart what must be separate, and the diagram begins to draw the plan. The same structure could run a farm: fields as nodes, edges weighted by shared water, machinery routes, and rotation sequence. For my project direction, this assignment's graph is a rehearsal — the next step is letting the relational structure output geometry.
